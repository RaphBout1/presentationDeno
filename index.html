Deno

Ryan Dahl

  • Préhistoire

    Seuls les navigateurs compilent le Javascript permettant de rendre dynamiques et interactives des pages web statiques avec des animations ou autres...

  • 2009

    Le monsieur sur la gauche à la brillante idée de prendre le moteur de compilation V8 Javascript utilisé par les navigateurs pour s’en servir en dehors et coder avec un noyau en C++ le runtime NodeJS

  • Années suivantes

    NodeJS est fréquemment utilisé pour créer des services côté serveur comme des API.
    Il permet aux développeurs Web d'utiliser un seul langage dans leurs projets en évitant PHP ou JAVA…
    Sa popularité grandit vite car il améliore la rapidité d'exécution. Il est beaucoup moins lourd et plus simple d'apprentissage pour des débutants.

  • Années suivantes

    Le gestionnaire de paquets npm fourni par défaut, permet facilment d'installer de nombreuses dépendances à plein de paquets.
    Tous ces facteurs font qu'aujourd'hui NodeJS est utilisé par beaucoup de grandes marques comme Netflix, Amazon, Paypal ou Linkedin

  • 2018

    Lors de la conférence ConfJS, Ryan Dahl fait part de ses 10 regrets par rapport à NodeJS concluant par la création du runtime DENO (oui c’est rigolo c’est NODE en verlan).
    Il a beaucoup de similitudes avec son cousin Node : son fonctionnement avec le moteur V8 ou son utilisation côté serveur de Javascript. Une des grandes différences est que Deno à un noyau codé en Rust

  • 2020...

    La première version stable de Deno est disponible (1.22.1). Il est possible de l’utiliser en production sans courir de risque. Une forte communauté s’est développée autour du projet et des mises à jour sont fréquentes. A l'heure où j’écris, la v1.29.1 vient de sortir (15/12/22).

INSTALLATION DE DENO

Rien de plus simple

Sous linux éxécuter curl -fsSL https://deno.land/x/install/install.sh | sh

Sous windows éxécuter irm https://deno.land/install.ps1 | iex

Puis tester votre installation avec deno --version

Si vous utilisez Visual Studio Code l'extension officielle est "Deno" de "denoland"

Assurez vous que le chemin vers Deno est bien mis dans les paramètres de Visual Studio Code

L'installation de NodeJs est un peu plus difficile mais Deno et Node se différencient surtout dans l'utilisation

GESTIONNAIRE DE PAQUETS

Décentralisé

import { pack } from "https://deno.land/x/pack.ts"

Frontière entre installation et éxécution abolie

Paquet importé depuis un URL puis mis en cache dans le disque dur (plus de node_module). Donc l'importation est unique et utilisable dans tous les projets.

Fléxible il est possible de créer ses propres paquets sans les publier

Bibliothéque officielle, vérifiée, bien fournie et mise souvent à jour. Possibilité d'utiliser npm dans les dernières versions.

NPM

npm i pack puis const pack = require ("pack").

Dépendant de npm, si on souhaite éxecuter votre code il faut d'abord télécharger toutes les dependances du package.json

Obligation d'installer toutes les dépendances sur chaque projet cela alourdit donc le disque dur

Très grande bibliothèque avec une communauté très active

IMPORTATION DE PAQUETS

Modules ES

import { pack } from "https://deno.land/x/pack.ts"

Chargement sélectif des parties utilisées à partir d'un paquet, c'est donc plus économique en mémoire

Chargement asynchrone, ce qui améliore la performance

CommonJS

npm i pack puis const pack = require ("pack").

Chargement synchrone, plus lent surtout s'il y à beaucoup de dépendances

Mises à jour plus compliquées et à faire manuellement

SUPPORT DE TYPESCRIPT

Compilateur TypeScript intégré

Comme Deno est un runtime à la fois Javascript et TypeScript, il suffit d'enregister un fichier en .ts au lieu du .js.

Impose une certaine rigueur de code donc un code plus robuste au final.

Processus très long

Installer TypeScript grâce à npm

npm install @types/node --save-dev

Mettre à jour les fichiers package.json, tsconfig.json

Puis pour finir être sûr que les modules prennent bien en charge @types.

SÉCURITÉ

Permissions simples à mettre en place :

  • > Lecture / Écriture du disque

    --allow-(read/write)=./dataPrivé.json

  • > Utilisations variables d'environnements

    --allow-env
  • > Liste de permissions

Pas de contrôle sur les accès des fichiers certains paquets installés peuvent accéder à des ressources privées.

PLEIN D'AUTRES PETITES CHOSES

Outil de formatage pour un code tout beau : deno fmt

Lire l'arborescence des dépendances de votre application simplement grâce à : deno info

Pour mettre à jour : deno upgrade Il n'offre par contre pas de possibilité de sélection de version pour l'instant

Module de test nativement (même si tester c'est douter) deno test :

Deno a nativement l'accès à l'API du navigateur ce qui permet de faire des appels asymchrones très facilement. Sans avoir besoin d'installer une librairie comme sur Node

POUR CONCLURE

Deno est prometteur et en beaucoup de points meilleur que Node. Seulement, il doit le prouver à la communauté. Pour l’instant Node a une avance considérable car des frameworks majeurs comme React ou Angular ne sont toujours pas adaptés officiellement sur Deno. Du fait de sa jeunesse, il évolue très vite car encore peu utilisé par des grosses entreprises en production. Les dernières mises à jour visent à améliorer la compatibilité avec NPM ce qui permettra peut-être de rapprocher Deno de son grand frère.

L'utilisation de Node est si importante qu'il parait difficle à Deno de le remplace totalement. L’idée de son créateur de reprendre tous les défauts de son projet pour repartir à zéro avec une meilleure expérience est très instructive.

Deno est une technologie à garder de vue.
